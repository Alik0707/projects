File Uploader Service принимает файлы через multipart/form-data, сохраняет их в локальное файловое хранилище, а метаданные загрузки (имя файла, размер, дата загрузки, статус обработки) фиксирует в базе данных.

Для обеспечения идемпотентности сервис проверяет наличие записи о файле в БД: если файл с таким же именем уже был успешно загружен, повторная загрузка не выполняется и клиенту возвращается ответ, что файл уже существует.

Таким образом предотвращается дублирование данных, а база используется как источник истины для контроля статуса и истории загрузок.
